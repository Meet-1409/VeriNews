# Screenshot Guide

The GitHub landing page should show the application that actually runs. Capture screenshots only after starting the app with:

```bash
streamlit run app.py
```

## Planned Captures

Store reviewed images under `docs/screenshots/` using stable, descriptive names:

- `analysis-workspace.png`: empty analysis page showing the primary workflow;
- `analysis-result.png`: a non-sensitive example with prediction, explanation, and related coverage; and
- `previous-searches.png`: local history view with sample data.

Do not capture API keys, personal queries, local file paths, private article text, or database contents that should not be public. Optimize images for GitHub before committing them.

The root README contains commented image placeholders. Uncomment them only after the corresponding files have been reviewed and added.
